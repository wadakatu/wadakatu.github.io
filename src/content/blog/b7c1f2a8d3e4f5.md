## はじめに

こんにちは、バックエンドエンジニアのwadakatuです。  
TPAC 2025 in Kobe のハッカソンにソロ参加し、1位になりました。  
当日の概要と作ったものについて、事実ベースでまとめます。

公式ページ: https://www.w3.org/2025/11/TPAC/hackathon/

## ハッカソン概要

- 日時: 2025-11-11 19:00–22:30 (JST)
- 会場: International Conference Center, Kobe (Room 501/502)
- チャレンジ:
  - Challenge #1 CSS Playground
  - Challenge #2 Dashboard（私はこちらを選びました）
- ルール: 90分で制作、ソロまたは2人チーム、AIツール利用可
- 提出形式: GitHub Issueにタイトル / 説明 / リポジトリ / スクショを投稿

---

## 参加経緯

ハッカソンがあることは知っていましたが、最初はあまり参加するつもりはありませんでした。
勝てる自信も自分ではなかったので。
ただ、私の会社（Studio, Inc）は今年初めてW3Cに参画しました。
Web Fontsの会議に参加できただけで目標は達成でしたが、自分としては何かもう少し爪痕を残したくなりました。
そこで、ハッカソン前日に深夜テンションで参加申し込みしました。

---

## 当日の様子

前日参加を決めたものの、当日は制作時間が限られていること、そして自分の強みを活かしたいと思っていました。  
その結果、デザインに自信がないので「CSS Playground」は見送り、まだ勝ち目がありそうな**ダッシュボード制作（Dashboard）に集中することにしました**。
ソロ参加で、Claude Code を使って Vibe Coding で制作しました。

90分という短い時間だったので、難しいライブラリは使わず、HTML / CSS / JavaScript の vanilla だけで書く方針にしました。  
デザインを言葉だけで伝えるのは難しかったため、Matrix と Fallout の Pip-Boy の参考画像を渡し、できるだけ早く意図を共有するようにしました。

やりたいことは山ほどありましたが、時間が限られていたため、できるだけシンプルな仕様にすることを優先しました。  
余談ですが、API連携がうまくいかなかった箇所があり、グラフの一部にはダミーデータを混ぜています。  
時間があれば修正したいポイントです。

## 作ったもの

作品名は「Matrix Terminal Dashboard」です。  
W3Cのデータを可視化する、レトロなターミナル風ダッシュボードを作りました。

- デモ: https://wadakatu.github.io/w3c-hackathon/
- リポジトリ: https://github.com/wadakatu/w3c-hackathon

### 作品の要点

- W3C API (https://api.w3.org) を使ってデータを取得
- 端末風UIで、グループ種別（WG / IG / CG など）や仕様ステータス（REC / WD / CR / NOTE など）を入力して可視化
- 1989年のCRT端末をイメージした、グリーンオンブラックのデザイン
- HTML / CSS / JavaScript で実装
- Claude Code を使って Vibe Coding で制作

## 投票と結果

投票は、ハッカソン参加者全員によるスタンプ投票でした。  
参加者のプロダクトは GitHub Issue に投稿され、そこにリアクションを付ける形式です。  
投票先の Issue は以下です。

https://github.com/w3c/hackathon-tpac-2025/issues/7

結果として、「Matrix Terminal Dashboard」が 1st Place Winner として掲載されました。  
公式ページの Submissions にも掲載されています。

https://www.w3.org/2025/11/TPAC/hackathon/

デモは、主催者の方のPCからモニターに映してもらう方式でした。  
作った人たちが前に出て発表する形式で、発表はすべて英語でした。

私の発表では、フレームワークやライブラリを使用していないこと、W3Cの標準機能を使っていることを伝えました。  
緊張はあまりなく、言いたいことは言えたと思います。
